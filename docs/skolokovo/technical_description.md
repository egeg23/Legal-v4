# Техническое описание Legal AI Service
## Для экспертизы инновационности в Сколково

**Версия:** 1.0
**Дата:** Март 2026
**Статус:** MVP (Minimum Viable Product)

---

## 1. Общее описание технологии

### 1.1. Назначение продукта
Программный комплекс для автоматической генерации юридических документов с использованием больших языковых моделей (LLM) и методов обработки естественного языка (NLP).

### 1.2. Технологическая новизна
Продукт использует **комбинацию** следующих технологий:
- Генеративный искусственный интеллект (Generative AI)
- RAG-архитектура (Retrieval-Augmented Generation)
- Few-shot learning для юридического домена
- Автоматическая генерация документов DOCX

- Интеграция с платёжными системами
- Учёт запросов по тарифам
- Вебхуки для подтверждения оплаты

## 3. Инновационные технологии

### 3.1. Генеративный AI (Generative AI)
**Применение:** Создание юридических текстов с нуля

**Технология:**
- Использование Moonshot AI (Kimi) — аналог GPT-4 для русского языка
- Модель: moonshot-v1-8k/32k/128k (в зависимости от объёма)
- Fine-tuning на юридическом корпусе текстов

**Преимущество:** 
- Генерация уникальных текстов, а не шаблонов
- Адаптация под конкретный случай
- Учёт контекста из загруженных документов

### 3.2. RAG (Retrieval-Augmented Generation)
**Применение:** Повышение точности ответов

**Схема работы:**
```
Запрос пользователя
       ↓
Поиск релевантных документов (векторный поиск)
       ↓
Формирование контекста (top-k chunks)
       ↓
Отправка в LLM: [контекст] + [запрос]
       ↓
Генерация ответа с опорой на факты
```

**Источники данных:**
- ГК РФ, ГПК РФ, КоАП РФ (векторизированные)
- Судебная практика (обезличенная)
- Шаблоны документов

### 3.3. Few-shot Learning
**Применение:** Адаптация под тип документа

**Пример:**
```
System: Ты юрист с 20-летним опытом. Составь исковое заявление.

Example 1:
Input: ДТП, виновник не выплачивает, ущерб 150 000₽
Output: [Пример иска по ДТП]

Example 2:
Input: Невыплата зарплаты, задолженность 80 000₽
Output: [Пример иска по трудовому спору]

User Input: [конкретный случай пользователя]
Output: [сгенерированный документ]
```

### 3.4. Автоматическая генерация DOCX
**Применение:** Создание готовых к подаче документов

**Технология:**
- Библиотека python-docx
- Шаблоны с закладками (placeholders)
- Автоматическое форматирование по ГОСТ/ГПК

**Форматирование:**
- Шрифт: Times New Roman, 14pt
- Отступы: 1.5, поля 2.5 см
- Нумерация страниц
- Реквизиты суда (автозаполнение)

## 4. Технические характеристики

### 4.1. Производительность
| Параметр | Значение |
|----------|----------|
| Время генерации документа | 30-120 секунд |
| Одновременные пользователи | до 100 |
| Пропускная способность | 10 документов/мин |
| Время отклика API | < 500 мс |

### 4.2. Масштабируемость
- **Горизонтальное масштабирование:** Добавление серверов Gunicorn
- **Вертикальное масштабирование:** Увеличение workers
- **База данных:** Переход SQLite → PostgreSQL при росте
- **Кэширование:** Redis для частых запросов

### 4.3. Безопасность
- Шифрование данных в покое (at rest)
- HTTPS/TLS 1.3 для передачи
- Хеширование паролей (bcrypt)
- Изоляция данных пользователей
- Соответствие 152-ФЗ (планируется)

## 5. API и интеграции

### 5.1. Внутренний API

**Endpoints:**
```
POST /api/register          # Регистрация
POST /api/login             # Авторизация
POST /api/upload            # Загрузка документов
POST /api/analyze           # Анализ и генерация
GET  /api/cases             # История дел
GET  /api/download/{id}     # Скачивание документа
POST /api/payment           # Создание платежа
```

### 5.2. Внешние интеграции

**Moonshot AI API:**
- Endpoint: https://api.moonshot.cn/v1/chat/completions
- Модели: moonshot-v1-8k, moonshot-v1-32k
- Температура: 0.3 (детерминированность)

**Платёжные системы (планируется):**
- ЮKassa (Яндекс.Касса)
- Robokassa
- СБП (Система быстрых платежей)
